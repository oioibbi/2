# 设计稿：木鱼打罪（打砖块 + 无尽模式）

## 目标

做一款“市面上主流打砖块手感 + 可持续成长”的网页版打砖块：

- 核心循环：击打罪砖 → 转化功德（本关累计）→ **通关入账** → 提升难度/进入下一关
- 两种模式：**闯关**（难度梯度合理）与 **无尽**（随机砖块永远打不完，难度随时间/波次提升）
- 主题改造：
  - 小球 = **圆形木鱼**
  - 砖块 = **罪孽值**（每块罪孽值不同）
  - 砖块越难打碎/越难命中（更高 HP、更快移动、更小体积、更危险位置）→ 罪孽值越高
  - 打碎砖块：消灭等量罪孽值 → 生成等量功德值（先记“本关功德”，通关后入账）

## 关键规则

### 功德的“入账”机制（你要求的核心）

- 每个小关都有两个计数：
  - `本关功德`：本关内打碎砖块得到（未入账）
  - `已入账功德`：通关后从 `本关功德` 一次性转入
- **失败（木鱼落地/失去生命）**：
  - `本关功德` 清零
  - 重试本关（砖块布局回到本关初始状态）
- **通关（清空本关砖块）**：
  - `本关功德` → 全部入账（永久不清零）
  - 进入下一关/下一波次

> 这样保证“用户获得过的功德值不清零”，同时也严格满足“只有通过一个小关才能获得在本关中累计的全部功德”。

### 罪孽值计算（非固定）

每块砖生成时决定：

- `HP`（需要击碎次数）
- `移动类型`（静止/水平/垂直）
- `移动速度`（像素/秒）

罪孽值示例公式（便于解释与平衡）：

```
罪孽值 = round( base * HP^1.2 * (1 + speed / 220) )
base = 10（可按关卡倍率提升）
```

含义：

- 更高 HP → 罪孽值显著更高
- 更快移动 → 罪孽值更高（“难命中”带来的难度收益）

### 玩法补强（参考主流打砖块常见“爽点”）

- 预瞄线：发射前显示大致反弹轨迹（降低挫败）
- 道具掉落（少量、可控）：扩展挡板/减速/额外功德（无尽更常见）
- 关卡梯度：先教会 → 再加变量（HP、密度、移动砖、速度、混合布局）

## 模式设计

### 闯关（Campaign）

设计原则：

- 1～3 关：静止砖 + 低速，练手感与角度控制
- 4～6 关：出现高 HP 砖，要求多次反弹与控球
- 7～9 关：引入移动砖（先慢后快），要求预判
- 10～12 关：混合布局 + 更高球速 + 更高砖密度

每关都能通关入账；每关本身就是你说的“小关”。

### 无尽（Endless）

结构：无限波次（每一波次也是一个“小关”）

- 波次生成：随机布局 + 难度权重（HP 分布、移动砖占比、速度倍率、砖密度）
- 难度提升：随 `波次` 与 `存活时间` 递增
- 计分/入账：每波清空 → 入账本波功德；失败 → 丢失本波功德并重试本波

## 记录与排行榜

- 永久累计功德：`totalMerit`（不清零）
- 模式内累计（单次游玩）：闯关/无尽分别统计
- 排行榜（本地）：分别记录两种模式的“单次游玩已入账功德 Top 10”
- 历史记录（本地）：每次游玩结束写入一条（时间、模式、到达关卡/波次、已入账功德、时长）

> 如需“全服排行榜”，需要后端与反作弊；当前实现默认本地榜。
